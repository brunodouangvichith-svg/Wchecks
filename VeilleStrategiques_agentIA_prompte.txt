=== VISION GLOBALE DU PROJET ===

Un agent qui surveille 11 dimensions liées au risque énergétique et 
géopolitique mondial, stockées dans Supabase, mises à jour automatiquement 
en arrière-plan sur Render, et visualisées sur une carte mondiale interactive.

1. RÉSERVE STRATÉGIQUE DE PÉTROLE US (SPR)
   - Source : API EIA (clé gratuite, https://www.eia.gov/opendata/register.php)
   - Série : WCSSTUS1, endpoint /petroleum/stoc/wstk/data/ (hebdomadaire)

2. PRIX DU BRENT CRUDE (spot)
   - Source : API EIA (même clé)
   - Série : RBRTE, endpoint /petroleum/pri/spt/data/ (quotidien)

3. PRODUCTION MONDIALE DE PÉTROLE ET GAZ NATUREL PAR RÉGION/PAYS
   - Source : API EIA, catégorie "international", endpoint /international/data/
   - Codes (activityId, productId, countryRegionId) à découvrir 
     dynamiquement via les facets de l'API avant d'écrire les requêtes 
     finales — pour le pétrole ET pour le gaz naturel (même endpoint, 
     mêmes découvertes de facets, productId différent)
   - Réutiliser un seul module client (eia_client.py) pour les deux 
     matières premières

4. CONFLITS GÉOLOCALISÉS PAR RÉGION
   - Source : API GDELT v2 (gratuite, sans clé)
   - Mots-clés : "oil", "pipeline", "OPEC", "sanctions", "war", "attack 
     on infrastructure"
   - Focus zones : Moyen-Orient, Russie/Mer Noire, Venezuela, Nigeria, 
     Mer Rouge
   - Récupère : date, titre, pays, lat/lon, ton, URL

5. DETTE PUBLIQUE DES PAYS (% du PIB)
   - Source : API World Bank (gratuite, sans clé), indicateur GC.DOD.TOTL.GD.ZS
   - Classification couleur : rouge >100% PIB, orange 50-100%, vert <50%

6. SITUATION ÉCONOMIQUE ET SOCIALE
   - Source : API World Bank — impôts (GC.TAX.TOTL.GD.ZS), chômage 
     (SL.UEM.TOTL.ZS), inflation (FP.CPI.TOTL.ZG)
   - Tensions sociales/manifestations : GDELT, mots-clés élargis 
     ("protest", "strike", "riot", "social unrest", "manifestation", 
     "grève générale"), géolocalisées par pays

7. BUDGET DÉFENSE ET ACTIVITÉ MILITAIRE
   - Budget défense (% PIB) : World Bank, MS.MIL.XPND.GD.ZS
   - Transferts d'armes : SIPRI Arms Transfers Database (CSV/Excel 
     gratuit à parser, pas d'API REST)
   - Proxy activité militaire : GDELT ("drone strike", "missile test", 
     "hypersonic missile", "military drill")
   ⚠️ LIMITE : aucune source fiable ne donne les stocks précis de 
   missiles/drones — proxies indirects uniquement, clairement identifiés

8. TRAFIC MARITIME MONDIAL (TANKERS)
   - Source : API AISstream.io (gratuite, clé requise)
   - Protocole WebSocket, mode "snapshot" (capture 30-60s)
   - Zones stratégiques : Ormuz, Malacca, Suez/Mer Rouge, Bab-el-Mandeb, 
     Golfe du Mexique
   - Filtre ship_type AIS 80-89 (tankers)
   - Récupère : MMSI, nom navire, lat/lon, cap, vitesse, horodatage
   ⚠️ LIMITES : pas de cargaison précise ; "going dark" possible en zone 
   de conflit

9. DÉCLARATIONS OFFICIELLES DES CHANCELLERIES/INSTITUTIONS
   - Source : agrégation de flux RSS officiels gratuits (ONU, Département 
     d'État US, Maison Blanche, Commission Européenne, OTAN, MAE français, 
     Kremlin en anglais — liste extensible)
   - Filtrage par mots-clés (sanctions, oil, energy, military, OPEC, 
     pipeline, strait, pays surveillés)
   - Complémentaire de GDELT (texte officiel brut vs couverture médiatique)
   ⚠️ LIMITE : couverture inégale, flux RSS individuellement fragiles

10. PRODUCTION INDUSTRIELLE PAR PAYS
    - Source : API World Bank (gratuite, sans clé), indicateur NV.IND.TOTL.ZS 
      (valeur ajoutée industrie, % du PIB, annuel)
    - Réutilise le module worldbank_client.py existant
    ⚠️ PRIX DE L'ÉNERGIE PAR PAYS — NON INTÉGRÉ :
    Aucune API gratuite fiable et à jour ne couvre les prix énergie 
    consommateur par pays (World Bank : indicateur discontinué ; IEA : 
    API complète payante, hors scope). Ne pas construire de proxy 
    approximatif donnant une fausse impression de précision. Documenter 
    explicitement cette absence dans le README (section "Limites connues").

11. PRODUCTION DE MINERAIS ET MÉTAUX STRATÉGIQUES PAR PAYS
    - Source : USGS National Minerals Information Center (gratuit, 
      PAS d'API REST — rapports annuels "Mineral Commodity Summaries" 
      en PDF/Excel à télécharger et parser)
      https://www.usgs.gov/centers/national-minerals-information-center
    - Matières premières prioritaires (configurables) : cuivre, lithium, 
      uranium, terres rares, or, cobalt
    - Données annuelles uniquement (rythme de publication USGS), pas de 
      mise à jour fréquente possible ni pertinente
    - Module dédié (minerals_client.py) sur le même principe que SIPRI 
      (dimension 7) : parsing de fichiers statiques stockés localement 
      (data/usgs/), pas de requête API
    ⚠️ LIMITE : format des rapports USGS pouvant varier d'une année à 
    l'autre — prévoir une tolérance dans le parsing, logger les lignes 
    non reconnues plutôt que planter

=== STOCKAGE : SUPABASE ===

Toutes les dimensions sont stockées dans Supabase (PostgreSQL managé, 
tier gratuit), mis en place dès la phase 1 comme fondation du projet.

- Projet Supabase gratuit (https://supabase.com) : 500 MB, pause après 
  7 jours d'inactivité (limite à documenter)
- SDK officiel `supabase-py`
- Variables d'environnement : SUPABASE_URL, SUPABASE_KEY

SCHÉMA DE BASE DE DONNÉES (une table par dimension) :
- spr_stocks (date, valeur_milliers_barils)
- brent_prices (date, prix_usd_baril)
- oil_production (pays_code, periode, valeur_barils_jour)
- gas_production (pays_code, periode, valeur_production_gaz)
- energy_conflicts (event_id unique, date, pays, lat, lon, titre, ton, url)
- social_tensions (event_id unique, date, pays, lat, lon, titre, ton, url)
- military_activity (event_id unique, date, pays, lat, lon, titre, ton, url)
- country_debt (pays_code, annee, dette_pct_pib)
- country_economy (pays_code, annee, impots_pct_pib, chomage_pct, inflation_pct)
- defense_budget (pays_code, annee, budget_pct_pib)
- arms_transfers (pays_exportateur, pays_importateur, annee, type_arme, quantite)
- maritime_traffic (mmsi, timestamp, lat, lon, vitesse, cap, zone_strategique)
- official_statements (url unique, date, institution, titre, extrait, langue)
- country_industry (pays_code, annee, production_industrielle_pct_pib)
- minerals_production (pays_code, annee, matiere_premiere, volume_tonnes, rang_mondial)
- risk_scores (pays_code, date_calcul, score_global, details_json)

Chaque table a une contrainte UNIQUE appropriée pour gérer l'anti-doublons 
via upsert (event_id, url, ou combinaison pays+date/matière selon la table).

MODULE CLIENT SUPABASE (supabase_client.py) :
- Fonctions génériques de connexion et d'upsert réutilisables par tous 
  les modules de collecte
- Fonctions de lecture pour le CLI (--latest, --history N)

=== MISE À JOUR AUTOMATIQUE EN ARRIÈRE-PLAN ===

- Scheduler (scheduler.py) via APScheduler, process persistant
- Fréquences différenciées et configurables (config.py) :
  - SPR : hebdomadaire | Brent : quotidien
  - Production pétrole/gaz mondiale : mensuel
  - Conflits/tensions sociales/activité militaire (GDELT) : toutes les 6h
  - Dette/économie/budget défense/industrie (World Bank) : mensuel
  - Trafic maritime (AISstream) : snapshot 4x/jour
  - Déclarations officielles (RSS) : toutes les 1-2h
  - Minerais stratégiques (USGS) : annuel / à la demande (--minerals-refresh)
  - Score de risque global : recalculé après chaque cycle, ou toutes les 6h
- Logging des succès/échecs par tâche (logs/scheduler.log), sans qu'un 
  échec bloque les autres sources
- Alertes en cas d'échecs répétés d'une source

=== DÉPLOIEMENT SUR RENDER ===

Le scheduler d'arrière-plan (scheduler.py) doit être déployé sur Render 
(https://render.com) en tant que Background Worker, pour tourner en 
continu indépendamment de toute machine locale.

- Type de service : Background Worker (tier gratuit disponible), PAS 
  un Web Service classique — un Background Worker n'attend pas de 
  requêtes HTTP, il exécute un process continu (adapté à APScheduler)
- Déploiement connecté à un repo GitHub : chaque push déclenche un 
  redéploiement automatique
- Commande de démarrage : `python scheduler.py`
- Variables d'environnement à configurer dans le dashboard Render (PAS 
  commitées dans le code) : EIA_API_KEY, SUPABASE_URL, SUPABASE_KEY, 
  AISSTREAM_API_KEY
- Fichier render.yaml (Infrastructure as Code, optionnel mais recommandé) 
  définissant le service Background Worker, la commande de démarrage, 
  et la liste des variables d'environnement requises (sans leurs valeurs)
- requirements.txt à jour avec toutes les dépendances du projet

⚠️ LIMITE DU TIER GRATUIT RENDER : les services gratuits peuvent être 
redémarrés périodiquement (redéploiements, maintenance) — le scheduler 
doit être résilient à un redémarrage (aucun état critique conservé 
uniquement en mémoire du process, tout ce qui doit persister va dans 
Supabase). Pas de garantie 24/7 parfaite, mais suffisant pour un outil 
de veille (pas un système critique temps réel).

Le CLI reste utilisable en local pour consultation ponctuelle, connecté 
à la même base Supabase que le scheduler déployé sur Render — les 
données collectées en arrière-plan sont immédiatement consultables en 
local sans redéploiement.

données sur les catastrophes naturelles (séismes, incendies, inondations, tempêtes, etc.), voici les meilleures options disponibles :

1. NASA EONET (Earth Observatory Natural Event Tracker)
L'API EONET de la NASA est l'une des plus populaires et des plus simples à utiliser pour suivre les événements naturels en cours ou récents à l'échelle mondiale.
Types de catastrophes : Feux de forêt, éruptions volcaniques, tempêtes sévères, glaces de mer, etc.
Format : REST (JSON / GeoJSON).
Accès : Gratuit et sans clé d'API (ou avec une clé gratuite d'API NASA pour des quotas plus élevés).
Exemple d'URL : * Tous les événements actifs : https://eonet.gsfc.nasa.gov/api/v3/events
Par catégorie (ex: volcans) : https://eonet.gsfc.nasa.gov/api/v3/categories/volcanoes
Documentation : eonet.gsfc.nasa.gov

2. GDACS (Global Disaster Alert and Coordination System)
Le GDACS est le système officiel de coordination et d'alerte géré conjointement par les Nations Unies et la Commission européenne. C'est l'API de référence pour la gestion de crise en temps réel.
Types de catastrophes : Séismes, inondations, cyclones tropicaux, sécheresses, éruptions volcaniques.
Format : REST (GeoJSON, XML, RSS).
Accès : Gratuit, sans inscription.
Exemple d'URL : * Obtenir les séismes de niveau d'alerte rouge : https://www.gdacs.org/gdacsapi/api/events/geteventlist/SEARCH?eventlist=EQ&alertlevel=red
Documentation : GDACS API Quickstart
